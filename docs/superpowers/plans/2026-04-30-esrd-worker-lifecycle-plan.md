> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

### Phase 3 — Python token guards

**Critical placement note (added by review pass 1)**: existing tests do `from feishu_adapter_runner.__main__ import main`. A top-of-module guard runs at import time and breaks them. The guard MUST live inside the `if __name__ == "__main__":` block (or at the top of the `main()` callable). `python -m feishu_adapter_runner` sets `__name__ == "__main__"` so the guard fires for that path; tests that import `main` directly bypass it (they're in-process — operator can't accidentally trigger via `uv run`).

- [ ] **3.1** Add guard to `py/src/feishu_adapter_runner/__main__.py` **inside the `if __name__ == "__main__":` block** (or at the top of the `main()` callable, whichever is the file's existing structure):
  ```python
  if __name__ == "__main__":
      import os, sys
      if not os.environ.get("ESR_SPAWN_TOKEN"):
          sys.stderr.write(
              "feishu_adapter_runner: must be spawned by esrd via erlexec; "
              "manual `uv run` invocation is unsupported (would create orphan "
              "adapter competing for Feishu app credentials).\n"
              "To debug locally:\n"
              "  esr daemon stop\n"
              "  ESR_SPAWN_TOKEN=__debug__ uv run --project py python -m "
              "feishu_adapter_runner ...\n"
          )
          sys.exit(2)
      main()
  ```
- [ ] **3.2** Same shape in `py/src/cc_adapter_runner/__main__.py` (replace `feishu_adapter_runner` with `cc_adapter_runner` in error text).
- [ ] **3.3** Same shape in `py/src/generic_adapter_runner/__main__.py`.
- [ ] **3.4** Same shape in `py/src/esr/ipc/handler_worker.py` (find the existing `if __name__ == "__main__":` block; prepend the guard before the existing entry-point logic).
- [ ] **3.5** Write `py/tests/test_spawn_token_guard.py`:

  - Test: same with random token like `"abc123"` → past the guard
  - **Run for all 4 entry points** (parameterize the test by module name).

## Open issues

- **`cleanup_orphans` API removal coordination with running prod**: prod operators have muscle memory for `esr daemon doctor --cleanup-orphans`. The flag-removal phase (Phase 4) emits a deprecation warning for one PR cycle? Spec says delete cleanly; lean toward delete (operators learn fast, no upgrade-window users).
- **erlexec `:kill_timeout` default vs adapter-friendly 10s**: spec recommends 10s for adapters. Default in `Esr.OSProcess` may be 5s. Phase 2.2 should set `kill_timeout: 10_000` in AdapterProcess's erlexec init opts.
